# Phase 2 Proof of Concept — "Daily Qur'an" Preview

**Date:** 2026-06-19
**Status:** Draft for review
**Lives in:** `tqe-landing` repo, new route `/daily`; linked from the Phase 2 card with a "Preview" badge.

## Goal

A small, polished, interactive taster of **Phase 2 (Daily Qur'an Challenge)** to show
alongside the landing page — turning the abstract "three-phase vision" into something a
donor can *try*. Demo-first, but with a **real email capture** so outreach traffic
converts to subscribers. (Decision: option **(c)** — demo + real capture.)

## Scope

- **Three "days"**, click between them, mapped to the first three mushaf pages:
  - **Day 1 = al-Fātiḥa (1:1–7)**
  - **Day 2 = al-Baqara 1–5**
  - **Day 3 = al-Baqara 6–16**
  - Exact verse ranges come from the authoritative mapping CSV (see Assets).
- Clearly badged **"Preview."** Not the full daily pipeline; no scheduling/rotation.

## The daily-page experience

- Web-native reading view styled to **evoke the book** (Scheherazade Arabic, EB Garamond,
  navy/brass/cream) — presentation option **(c)**.
- **Sequenced player** walks: `AR1 → EN1 → AR2 → EN2 → … → commentary`.
  - The currently-sounding verse is **highlighted**; its translation is shown/emphasised
    in step. Segment-level sync (one audio file per segment) — no word-level timestamps.
  - Tap any verse to jump to it. Play/pause.
- **Commentary** (text + audio) plays at the end of the page.

## Audio

- **Arabic recitation:** Husary **"Muallim"** (slow teaching style), Quran.com API
  recitation **id 12**; per-āyah mp3s (`…/Husary_Muallim_128kbps/SSSAAA.mp3`), fetched at
  build time and cached in `assets/`.
- **English verse translation:** **already generated** for the whole Qur'an in the "Frank"
  voice (reuse from TSQ — see Assets). Display text = the matching **revised/modified
  Qarai** wording, so on-screen text and audio agree.
- **Commentary narration:** generate fresh with **ElevenLabs v3**, a **new British male,
  educated-sounding voice** (auditioning **George** `JBFqnCBsd6RMkjVDRZzb` vs **Daniel**
  `onwK4e9ZLuTAKqWW03F9` — decision pending). Script follows
  `docs/commentary-narration-style.md` (Arabic terms in Arabic script; numbers as words).
- **Tight pacing (change from TSQ):** fragments carry ~0.9s baked silence each. Trim
  leading/trailing silence at build time (`ffmpeg silenceremove`); player adds a small
  **configurable inter-fragment gap** (default ~200ms).

## Data

One small JSON per day: ordered list of verses `{ ar, en, ar_audio, en_audio }` plus
`{ commentary_text, commentary_audio }`. Generated by a build script from the CSV sources
+ commentary files; consumed by the static page.

## Email capture (the real part)

- Simple "Get the daily Qur'an in your inbox" form on `/daily`.
- No actual scheduled sends in the POC — just capture (clearly framed as "early access").
